Component Library

A comprehensive showcase of all UI components available in the Therapy Docs application. All components follow our design system principles: consistency, clarity, calm, accessibility, and simplicity.

How to use: Click on any component to copy its HTML code to your clipboard.

Keep in sync: This file must stay synchronized with docs/COMPONENTS.md. When adding or modifying components, update both files. Also update the CSS index at the top of styles/input.css.

With Collapsible Options

Collapsible options (CBT, DBT, Psychodynamic) appear in their defined positions when expanded. Collapsed: All, Humanistic, Family. Expanded: All, CBT, DBT, Humanistic, Psychodynamic, Family.

Selected: | Visible options: 

<div x-data="{
  selected: 'all',
  expanded: false,
  sliderStyle: '',
  options: [
    { id: 'cbt', label: 'CBT', icon: 'approach-cbt', collapsible: true },
    // ... more options with collapsible: true/false
  ],
  get visibleOptions() {
    return this.expanded ? this.options : this.options.filter(o => !o.collapsible);
  },
  updateSlider() {
    this.$nextTick(() => {
      const container = this.$refs.control;
      const btn = container?.querySelector(`[data-option-id='${this.selected}']`);
      if (container && btn) {
        const cRect = container.getBoundingClientRect();
        const bRect = btn.getBoundingClientRect();
        this.sliderStyle = `width: ${bRect.width}px; transform: translateX(${bRect.left - cRect.left - 4}px)`;
      }
    });
  },
  selectOption(id) { this.selected = id; this.updateSlider(); },
  toggle() {
    this.expanded = !this.expanded;
    if (!this.expanded && this.options.find(o => o.id === this.selected)?.collapsible) {
      this.selected = 'all';
    }
    this.updateSlider();
  },
  init() { this.updateSlider(); }
}">
  <div class="segmented-control relative" x-ref="control">
    <div class="absolute top-1 bottom-1 left-1 bg-white rounded-md shadow-sm transition-all duration-200"
         :style="sliderStyle"></div>
    <button class="segmented-control-option ..." @click="toggle()">
      <svg class="icon"><use :href="expanded ? '...#show-options-from-left' : '...#hide-options-from-left'"></use></svg>
    </button>
    <template x-for="option in visibleOptions" :key="option.id">
      <button :data-option-id="option.id" @click="selectOption(option.id)">...</button>
    </template>
  </div>
</div>

Sub-settings (Nested/Conditional)

When a parent toggle reveals child settings, use indentation with a left border to show hierarchy. Use <template x-if> to conditionally render the sub-settings. 

<!-- Parent Toggle -->
<label class="flex items-center justify-between p-3 bg-neutral-50 rounded-lg cursor-pointer hover:bg-neutral-100 transition-colors">
  <!-- ... parent toggle content ... -->
</label>

<!-- Sub-settings (conditional) -->
<template x-if="parentEnabled">
  <div class="ml-8 space-y-3 border-l-2 border-neutral-200 pl-4">
    <!-- Number input sub-setting -->
    <div class="flex items-center justify-between p-3 bg-neutral-50 rounded-lg">
      <!-- ... -->
    </div>

    <!-- Toggle sub-setting -->
    <label class="flex items-center justify-between p-3 bg-neutral-50 rounded-lg cursor-pointer ...">
      <!-- ... -->
    </label>
  </div>
</template>
